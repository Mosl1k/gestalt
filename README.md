Список покупок или того, что надо не забыть.
main.go - бекэнд сайта с api, работает в связке с redis.
alice.py - flask приложение для навыков Алисы, умеет добавить в определённый список, или прочитать всё из списка "купить/не забыть".
Всё это дело разворачивается через docker-compose, а
второй компоуз чисто для репликации редиса.


Go - Backend API (обработка запросов, логика сервиса). 
Python (Flask) - Дополнительное приложение — alice.py, для интеграции с голосовым помощником. 
Docker / Docker-Compose - Контейнеризация сервиса, оркестрация: запуск backend + redis + опции — облегчает деплой и масштабирование, плюс есть вариант с репликацией Redis. 
Redis - В качестве хранилища данных (списки покупок / задач). 
HTML + Frontend - веб-интерфейс для взаимодействия со списком. 
Многокомпонентная архитектура + микро-сервисный подход 

Комбинация различных частях: API, хранилище, возможные интеграции — даёт представление о разделении ответственности и модульности.
